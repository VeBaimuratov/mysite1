# BUG REPORT — Mysite1

Дата: 2026-03-06
Тестировщик: Claude (ручное + статический анализ кода)
Файлы: Mysite1.html, Mysite1.css, Mysite1.js

---

## P1 — КРИТИЧЕСКИЕ (блокеры)

### BUG-001: Страница "О студии" не адаптируется на мобильных
- **Файл:** Mysite1.css:513-517
- **Описание:** `.about-page-layout` всегда остаётся `grid-template-columns: 1fr 1fr`. В responsive-блоках (Mysite1.css:1019-1023) исправляются `.about-grid` и `.order-inner`, но не `.about-page-layout`. На экранах <768px две колонки сжимаются до нечитаемого состояния — текст налезает, изображение схлопывается.
- **Ожидаемое:** На мобильных `.about-page-layout` должен переключаться в одну колонку.
- **Приоритет:** P1 (блокер)

### BUG-002: У полей формы нет атрибута `name`
- **Файл:** Mysite1.html:244-271
- **Описание:** Все `<input>`, `<select>`, `<textarea>` имеют только `id`, но не `name`. Без `name` данные не попадут в `FormData` и не передадутся на сервер при реальной отправке.
- **Затронутые поля:** `name`, `phone`, `email`, `object`, `area`, `message`
- **Приоритет:** P1 (блокер)

### BUG-003: Конфликт имён в JS — `lbClose` / `lbClose_()`
- **Файл:** Mysite1.js:144, Mysite1.js:171
- **Описание:** Переменная `const lbClose = document.getElementById('lbClose')` и функция `function lbClose_()` — подчёркивание добавлено как обход конфликта имён, а не решение. При рефакторинге легко допустить ошибку, вызвав `lbClose()` вместо `lbClose_()`.
- **Приоритет:** P1 (критичный дефект кода)

### BUG-004: Пустой экран при прямой навигации (пропуск страниц)
- **Файл:** Mysite1.js:17-29
- **Описание:** `goToPage()` при движении вперёд ставит только `pages[next].style.transform = 'translateY(0)'`, при движении назад — `pages[current].style.transform = 'translateY(100%)'`. Если промежуточная страница не посещалась, она остаётся за экраном.
- **Воспроизведение:**
  1. Главная (стр. 0)
  2. Клик "Портфолио" (стр. 2) — переход напрямую
  3. Клик точки стр. 1 ("О студии")
  4. Результат: пустой экран (стр. 1 всё ещё `translateY(100%)`)
- **Приоритет:** P1 (блокер навигации)

### BUG-005: `pointer-events: all` — невалидное CSS-значение
- **Файл:** Mysite1.css:655
- **Описание:** `.lightbox.open` использует `pointer-events: all`. Для HTML-элементов валидное значение — `auto` (`all` работает только в SVG). В части браузеров лайтбокс может не реагировать на клики.
- **Исправление:** Заменить на `pointer-events: auto`.
- **Приоритет:** P1

### BUG-006: `<img src="">` в лайтбоксе при загрузке страницы
- **Файл:** Mysite1.html:296
- **Описание:** `<img class="lb-image" id="lbImage" src="" alt="">` — пустой `src` вызывает лишний HTTP-запрос к текущему URL страницы в Chrome и Firefox.
- **Приоритет:** P1

## P2 — ВАЖНЫЕ

### BUG-007: Телефон и email без валидации
- **Файл:** Mysite1.html:247-252
- **Описание:** Поле телефона `type="tel"` без атрибута `pattern` — принимает любой текст ("abc"). Поле email не имеет `required` — можно отправить заявку вообще без контактных данных (ни телефона в корректном формате, ни email).
- **Ожидаемое:** `pattern` для телефона, `required` для email или хотя бы одного контакта.
- **Приоритет:** P2

### BUG-008: Мобильное меню не закрывается кликом вне его
- **Файл:** Mysite1.js:111-113
- **Описание:** Бургер-кнопка открывает `.nav.open`, но клик по любой области вне меню его не закрывает. Пользователь должен снова найти и нажать бургер.
- **Ожидаемое:** Клик вне меню или по оверлею закрывает его.
- **Приоритет:** P2

### BUG-009: Свайп на мобильном не проверяет состояние лайтбокса
- **Файл:** Mysite1.js:90-100
- **Описание:** Обработчик `wheel` (строка 54) проверяет `lightbox.classList.contains('open')`, но `touchend` — нет. При свайпе в открытом лайтбоксе происходит переход между страницами.
- **Приоритет:** P2

### BUG-010: Нет `-webkit-backdrop-filter` для Safari
- **Файл:** Mysite1.css:126
- **Описание:** `backdrop-filter: blur(10px)` без `-webkit-` префикса. На Safari 9-14 и старых iOS шапка теряет эффект размытия.
- **Приоритет:** P2

## P3 — UX / ПОВЕДЕНИЕ

### BUG-011: `wheelBuffer` не сбрасывается при смене направления
- **Файл:** Mysite1.js:49-73
- **Описание:** Если прокрутить вниз (накопить буфер ~60 при пороге 80), затем резко вверх — буфер не обнуляется. Значение медленно проходит через ноль, вызывая "задержанный" или нечувствительный переход страниц.
- **Приоритет:** P3

### BUG-012: Мобильное меню всегда тёмное на светлых страницах
- **Файл:** Mysite1.css:1028-1038
- **Описание:** `.nav.open` имеет фиксированный `background: var(--dark)`. На страницах 1-4 шапка переключается в `.header.light`, но выпадающее меню остаётся чёрным. Визуальное несоответствие.
- **Приоритет:** P3

### BUG-013: Лайтбокс не перехватывает фокус (focus trap)
- **Файл:** Mysite1.js:165-169
- **Описание:** При открытом лайтбоксе клавиша Tab позволяет фокусу уйти на элементы позади оверлея (навигация, форма). Нет focus trap.
- **Приоритет:** P3

### BUG-014: Нет ESC для закрытия модального окна успеха
- **Файл:** Mysite1.js:218-220
- **Описание:** Лайтбокс закрывается по Escape, модалка успеха — нет. Закрыть можно только кнопкой "Хорошо".
- **Приоритет:** P3

### BUG-015: Кнопки без подписей для скринридеров (a11y)
- **Файл:** Mysite1.html:22, 292-294, 28-32
- **Описание:**
  - Бургер (строка 22): только символ `&#9776;`, нет `aria-label="Открыть меню"`
  - Кнопки лайтбокса (строки 292-294): нет `aria-label` ("Закрыть", "Назад", "Вперёд")
  - Точки навигации (строки 28-32): кликабельные `div.dot` без `role="button"`, `tabindex="0"`, `aria-label`
- **Приоритет:** P3

### BUG-016: Страница 2 (Портфолио) не в `noScrollPages`
- **Файл:** Mysite1.js:15
- **Описание:** `const noScrollPages = [1, 3, 4]` — страницы 0 и 2 отсутствуют. Если контент портфолио переполняет viewport (маленький экран/ноутбук), колесо мыши сначала прокручивает внутренний контент, а не переключает страницу. Пользователь "застревает".
- **Приоритет:** P3

### BUG-017: Отсутствует CSS-правило для `.about-img-wrap`
- **Файл:** Mysite1.css (отсутствует), Mysite1.html:53
- **Описание:** В HTML есть `<div class="about-img-wrap">`, но в CSS нет правила `.about-img-wrap`. Есть только `.order-img-wrap`. Изображение внутри использует `height: 100%`, но родитель без явной высоты — изображение может не заполнять левую колонку полностью.
- **Приоритет:** P3

## P4 — КОНТЕНТ / МЕЛОЧИ

### BUG-018: Заглушки в футере
- **Файл:** Mysite1.html:282-285
- **Описание:** Телефон `+7 (495) 000-00-00` и email `hello@forma-studio.ru` дублируются дважды — в `footer-inner` (строки 282-283) и в `footer-copy` (строка 285). Информация повторяется, а номер явно заглушечный.
- **Приоритет:** P4

### BUG-019: Опции `<select>` без атрибута `value`
- **Файл:** Mysite1.html:258-263
- **Описание:** `<option>Квартира</option>` и т.д. — без `value`. При сериализации формы вернётся текст на русском вместо машиночитаемого значения.
- **Приоритет:** P4

### BUG-020: Два `keydown`-обработчика на document
- **Файл:** Mysite1.js:77, 204
- **Описание:** Два отдельных `addEventListener('keydown')`. Работает за счёт раннего return, но дублирование логики и хрупкая архитектура.
- **Приоритет:** P4

### BUG-021: Мёртвые CSS-классы
- **Файл:** Mysite1.css:392, 413, 419, 425, 458
- **Описание:** Стили объявлены, но не используются ни в одном HTML-элементе: `.scroll-hint`, `.about`, `.about-grid`, `.about-text`, `.services`.
- **Приоритет:** P4

### BUG-022: Несогласованные цвета акцента
- **Файл:** Mysite1.css:209 vs css:16
- **Описание:** `--accent: #c8a96e` (тёплое золото), но `.nav-btn` использует захардкоженный `#e6c84a` (яркий жёлтый). Два разных оттенка в одной теме.
- **Приоритет:** P4
