# DEVOPS.md — Дорожная карта проекта "Серёга"

---

## Где мы сейчас

Сайт-портфолио студии дизайна интерьера "Серёга" полностью разработан, протестирован и упакован в Docker.
Текущий статус: **готов к деплою**.

Стек: HTML5 + CSS3 + Vanilla JS, без фреймворков и сборщиков.
Тестирование: Playwright, 576/576 тестов проходят на 8 конфигурациях браузеров и устройств.
Деплой: Docker + nginx.

---

## Что было сделано и зачем

### Этап 1: Разработка сайта

Создан fullpage-сайт без фреймворков (React, Vue и т.д.) — потому что сайт статический,
контент не меняется динамически, серверной логики нет. Vanilla JS + CSS достаточно для
такой задачи, а отсутствие фреймворков означает:
- нет зависимостей, которые устаревают
- нет сборщика (webpack, vite) — браузер открывает HTML напрямую
- быстрая загрузка (нет JS-бандла в сотни КБ)

Fullpage-навигация реализована через CSS `transform: translateY()` — самый
производительный способ анимации (не вызывает reflow, работает на GPU).

5 страниц: Главная (hero), О студии, Портфолио (лайтбокс), Команда, Форма заказа.

### Этап 2: Качество кода (27 багов)

Проведено 2 ручных аудита кода. Найдено 27 багов, все исправлены:

- **Критические (P1):** адаптивность About-страницы, отсутствие `name` у полей формы,
  конфликт имён в JS (`lbClose`), пустой экран при пропуске страниц, невалидный CSS,
  битый `<img src="">`
- **Важные (P2):** валидация телефона/email, мобильное меню, свайп в лайтбоксе, Safari prefix
- **UX (P3):** доступность (aria-label, role, tabindex, focus trap), keyboard navigation
- **Мелочи (P4):** дублирующиеся контакты в футере, select без value, мёртвый CSS

Зачем это важно: корректная работа формы (поле `name` — данные попадают на сервер),
доступность для людей с ограниченными возможностями (aria-атрибуты, focus trap),
кросс-браузерная совместимость (webkit-префиксы для Safari).

### Этап 3: Автотестирование (576 тестов)

Написано 72 теста Playwright, запускаемые на 8 конфигурациях:
3 desktop (Chrome, Firefox, Safari) + 2 tablet (iPad portrait/landscape) + 3 mobile.

Playwright выбран потому что:
- работает с реальными браузерами (Chromium, Firefox, WebKit) — не эмуляция
- headless-режим (без GUI) — подходит для CI/CD
- кросс-платформенный (Windows, Mac, Linux)
- `file://` URL — тесты открывают HTML напрямую, без сервера

17 групп тестов: структура DOM, навигация, лайтбокс, форма, мобильное меню,
доступность, адаптивность (5 брейкпоинтов), CSS-корректность.

Результат: 576/576 = зелёный статус на всех устройствах.

### Этап 4: Контейнеризация (Docker)

Сайт упакован в `nginx:alpine` контейнер. Nginx выбран потому что:
- оптимален для раздачи статических файлов (HTML/CSS/JS)
- встроенный gzip (уменьшает трафик)
- кэширование статики (CSS/JS кэшируется браузером на 30 дней)
- `alpine` образ весит ~25 МБ — минимальный размер

`docker-compose.yml` — запуск одной командой, порт 8080, автоперезапуск.
`.dockerignore` — исключает node_modules, тесты, документацию → образ остаётся маленьким.

## Что нужно сделать дальше

### Ближайшие задачи (Must have)

#### 1. Реальная отправка формы

Сейчас форма не отправляет данные — `e.preventDefault()` показывает модалку-заглушку.
Нужен backend или внешний сервис для обработки заявок.

Варианты (от простого к сложному):

| Вариант | Время | Описание |
|---------|-------|---------|
| **EmailJS** | ~1-2 ч | Отправка на email прямо из браузера без сервера |
| **Formspree** | ~30 мин | Готовый сервис: добавить `action` к `<form>` |
| **Telegram Bot** | ~2-3 ч | Заявки приходят в Telegram |
| **Node.js/Python backend** | ~1-2 дня | Полный контроль, сохранение в БД |

Рекомендация: начать с Formspree или EmailJS — быстро и без сервера.

#### 2. Деплой (публичный URL)

Сейчас сайт работает только локально на `localhost:8080`.

Варианты:

| Вариант | Время | Стоимость | Описание |
|---------|-------|-----------|---------|
| **Netlify** | 5 мин | Бесплатно | Drag & drop папки → публичный URL |
| **GitHub Pages** | 15 мин | Бесплатно | Автодеплой из репозитория |
| **VPS + Docker** | 1-2 ч | ~5$/мес | Полный контроль, nginx уже настроен |

Рекомендация: Netlify — самый быстрый старт.

#### 3. CI/CD пайплайн

Сейчас тесты запускаются вручную: `npm test`.
Нужен автозапуск при каждом изменении кода.

GitHub Actions: при `git push` в `main` → запустить 576 тестов → если зелено → деплой.

Файл `.github/workflows/ci.yml`:
```yaml
name: CI

on:
  push:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
      - run: npx playwright install --with-deps
      - run: npm test
```

### Среднесрочные задачи (Nice to have)

#### 4. HTTPS

Если деплой на VPS — нужен SSL-сертификат для `https://`.
Let's Encrypt + Certbot: бесплатный SSL, автообновление каждые 90 дней.

#### 5. Мониторинг

После деплоя нужно знать что сайт работает.
Uptime Robot (бесплатно): пинг каждые 5 минут, алерт в Telegram если сайт упал.

#### 6. Аналитика

Яндекс Метрика или Google Analytics — узнать кто заходит, откуда, что смотрит.
Добавить 1 строку кода в `<head>` Mysite1.html.

#### 7. Реальные фотографии

Сейчас в портфолио — placeholder-фото с Unsplash.
Заменить на реальные фотографии проектов студии.
Оптимизировать: WebP формат, размер не более 800KB на фото.
